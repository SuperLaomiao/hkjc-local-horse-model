## 口径说明

这里分两个口径：

1. 赛前锁定快照：从 Git 历史里找当时 dashboard 已保存的赛前 forecast，再对照后来的官方赛果。这个最接近“当时真的给过的判断”，但样本只有 20 场，而且当时没有实时独赢赔率。
2. 滚动回测：模型按比赛时间顺序预测一场、结算一场、再学习下一场。样本更大，但它是模型体检，不等同于每场真实赛前已公开锁单。

## 判断

- 如果目标是“每场找一匹最可能赢”：当前模型不合格。近期头号马只有 11.0%，全量也只有 13.0%，明显输给市场热门。
- 如果目标是“少量价值候选”：历史上有过正 ROI，但近期已经回吐到 0.0%，且赛前锁定快照并不好。
- 模型偏爱中高赔率马，头号马平均赔率约 12 倍；这会带来偶尔大回报，但连续不中很正常。
- 当前最需要改进的是校准概率，不是继续喊“必胜”。模型概率看起来太容易把长赔率马抬高。

## 建议

1. 不再使用“必胜模式”这个目标词，改成“赛前概率候选 + 赛后校准”。
2. 暂停自动执行价值下注；下一轮只做纸上记录。
3. 每场保存赛前锁定 forecast 快照，包括生成时间、最低入场赔率、最终是否下注，避免赛后回测和真实预测混在一起。
4. 增加一个校准指标：模型给 15% 胜率的马，长期是否真的接近 15%。
5. 下一步优先优化“低赔率热门识别”和“长赔率误判过滤”，否则头号马胜率会继续偏低。
