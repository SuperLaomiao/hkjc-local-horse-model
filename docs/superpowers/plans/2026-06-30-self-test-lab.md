# Self-Test Lab Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn the static HK local horse model dashboard into a practical self-test lab where the user can lock model forecasts, record their own paper picks, and see model performance without confusing rolling backtests with live predictions.

**Architecture:** Keep the site static and GitHub Pages friendly. Add pure tested modules for model performance summaries and user paper-pick settlement, then wire them into the existing `app.js` UI using `localStorage` for browser-local user records. Extend dashboard JSON with a compact `performance` object generated by the model pipeline.

**Tech Stack:** Vanilla ES modules, Node.js built-in `node:test`, static HTML/CSS/JS, existing HKJC model CLI.
